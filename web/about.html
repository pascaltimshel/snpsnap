SNPsnap main usage

Match input SNPs
SNPsnap is designed to generate matched sets of SNPs that can be used to calibrate background expectations in SNP-based enrichment analysis.

Annotate input SNPs using 1000 Genome data
SNPsnap allows users to quickly retrieve 1000 Genome population specific annotation about loci bounderies, nearest genes, genes in loci, minor allele frequency and much more.

Tips & Tricks

Tip #1: fast SNP input annotation
SNPsnap main purpose is to match input SNPs. However, many users are often only interested in annotating their input SNPs using 1000 Genome population specific data. If you are only interested in annotatating your input SNPs, setting Number of matched SNPs to a low value (e.g. 1) will highly speed up the runtime for your job. Of course you should also make sure to enable Annotate input SNPs.

Tip #2: independence of SNPs (clumping)
SNPsnap has a nifty extra feature to check the independence of your input SNPs. This feature was originally designed to facilitate proper use of the SNP-matching results. However, many of our users find it useful on its own - just as a quick way to clump your SNPs using PLINK's algorithm.
